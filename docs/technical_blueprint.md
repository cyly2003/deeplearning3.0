# ECOTOX-QSAR 迁移学习技术蓝图

## 1. 任务理解

本项目目标是重构一个面向 ECOTOX 数据库的 QSAR/迁移学习系统。系统从已初步清洗的 SQLite 关系数据库出发，构建可追溯的建模宽表、条件聚合样本、多任务深度学习模型和训练控制 GUI。

核心预测目标是：当用户输入化合物分子结构，并指定物种、终点、效应水平、介质、暴露时长等条件时，模型能够预测对应毒性值；进一步支持多物种批量预测、SSD 风险评估和土壤污染生态风险分析。

## 2. 已确认关键决策

### 2.1 数据入口

- 主线从已初步清洗的 SQLite 数据库开始。
- 当前数据库以多张关系表为主，需要先构建建模宽表。
- 第一版不把从原始 ECOTOX 文件重建数据库作为主任务。

### 2.2 样本粒度

- 主模型使用条件聚合级样本。
- 记录级数据保留用于追溯、诊断、GUI 检查和敏感性分析。
- 条件聚合不粗暴跨暴露时长，只允许极小容差合并，例如 95.9 h、96 h、96.1 h。
- 模型中保留连续暴露时长，如 `duration_h` 和 `log1p_duration_h`。

### 2.3 目标变量构建

- 毒性值原始数据库尚未完成统一量纲和负对数转换。
- 主模型保留双尺度目标：
  - QSAR 主线优先摩尔尺度。
  - 风险评估输出转换回 `mg/L`、`mg/kg` 等常用单位。
- 土壤主任务直接预测 `-log10(mg/kg)`，服务土壤 SSD 和风险限值表达。
- 水相优先保留 `mol/L` 与 `mg/L`。
- 土壤/沉积物保留 `mg/kg`。
- 经口暴露质量剂量保留为辅助任务分支。

### 2.4 毒性值选择规则

- 忽略 `>`、`<`、`>=`、`<=` 等限定符。
- 优先使用 mean 毒性值。
- mean 缺失时：
  - 如果 min 和 max 同时存在，且剂量组数 `dose_group_count >= 3`，使用 `(min + max) / 2`。
  - 如果剂量组数小于 3，或 min/max 不完整，则剔除。
- 必须记录：
  - `tox_value_source`
  - `tox_value_imputed`
  - `tox_value_rule_id`
  - `excluded_reason`

### 2.5 缺失暴露时长

- 暴露时长缺失不直接剔除。
- 第一版采用“标准时长规则表 + 数据驱动众数补齐”。
- 优先按 endpoint、effect type、物种类群、介质、急慢性试验类型匹配人工规则。
- 匹配不到时，在相近分组中使用最常见暴露时长补齐。
- 必须记录：
  - `duration_imputed`
  - `duration_rule_id`
  - `duration_source`
  - `duration_confidence`
  - `duration_fill_method`

### 2.6 主任务与辅助任务

主任务：

- EC/LC
- NOEC
- LOEC

预测头粒度：

- 中等粒度。
- 按“终点家族 + 效应类型”建 head，例如：
  - `ECx_Mortality`
  - `ECx_Growth`
  - `ECx_Reproduction`
  - `NOEC_Growth`
  - `LOEC_Reproduction`
- `effect_level_x` 作为连续条件输入。

辅助任务：

- BAF/BCF 联合辅助学习。
- 经口暴露系统毒性辅助任务。

BAF/BCF 不作为主毒性任务输出，而作为共享知识来源。缺失 BAF/BCF 不强行插值，使用 mask loss，只对真实有标签的样本计算辅助损失。

经口辅助任务：

- 优先使用 `mg/kg body weight` 或 `mg/kg bw/day`。
- 允许将 `mg/individual`、`mg/organism` 通过物种成年体重换算为 `mg/kg bw`。
- 当前字段没有实验体重，第一版通过本地体重参考表或网络检索构建缓存，统一用物种成年体重换算。
- life stage 作为模型输入，让模型学习成年默认体重与生长阶段之间的交互。

### 2.7 分子和物种表征

第一版分子结构表征：

- RDKit 分子描述符。
- Morgan 指纹。
- 保留分子编码器插件接口，后续可接入 GNN、SMILES Transformer、ChemBERTa。

描述符权重机制：

- 延续描述符聚类权重逻辑。
- 每个小类内描述符权重和为 1。
- 每个大类下小类权重和为 1。

物种信息：

- 分类学层级 embedding。
- life stage、habitat、primary medium、exposure route、暴露时长等作为生态/实验上下文辅助输入。
- 第一版不强依赖外部 trait 数据库。

### 2.8 训练策略

- 多任务共享主干。
- 固定手动损失权重。
- 损失函数采用 Huber Loss。
- 主任务权重高于辅助任务。
- 本机可运行完整流程，但主要训练路径面向远程 GPU。
- 本机运行用于小样本调试、冒烟测试、配置验证、GUI 可用性检查。
- 远程训练接口第一版即预留。

### 2.9 数据划分

第一版全部执行，全部作为参考项：

- `random_split`
- `chemical_group_split`
- `species_group_split`
- `chemical_species_group_split`
- `medium_transfer_split`

### 2.10 评价指标

第一版使用常规回归指标：

- `R2`
- `RMSE`
- `MAE`
- `Huber loss`

按任务头、split、介质或任务域分别输出。

### 2.11 模型解释

第一版采用基础解释 + SHAP/Permutation 解释：

- 描述符小类权重。
- 描述符大类权重。
- Morgan 指纹重要性。
- 任务头相关权重或特征贡献。
- 分子描述符、物种分类学、暴露时长、life stage、介质等输入变量的 SHAP 或 permutation importance。

### 2.12 GUI

第一版 GUI 使用 PySide6 / Qt 桌面程序，目标是完整训练控制台：

- 实验配置页。
- 训练队列页。
- 实时日志页。
- 指标监控页。
- 输出管理页。
- 实验备注页。

执行位置：

- 工具可在本机运行。
- 训练主路径按远程 GPU 训练设计。
- 本机执行器保留用于调试和小规模试跑。

## 3. 推荐工作流程

1. 数据库字段扫描。
2. 字段映射配置。
3. 建模宽表构建。
4. 目标变量构建与单位换算。
5. 条件聚合样本构建。
6. 分子描述符与指纹计算。
7. 物种分类学与上下文编码。
8. BAF/BCF 辅助表构建。
9. 经口辅助任务表构建。
10. 多套 split 生成。
11. 训练配置生成。
12. 本机冒烟测试。
13. 远程训练。
14. 指标汇总。
15. 模型解释。
16. 后续 SSD/HC5/HC10 风险评估。

## 4. 科研风险提醒

- 单位换算失败记录不能静默删除，必须保留失败原因。
- mean 缺失后用 min/max 平均会引入不确定性，必须保留来源标记。
- 经口暴露辅助任务与环境暴露主任务量纲和机制不同，必须通过独立 head 和较低损失权重隔离。
- BAF/BCF 不能用全数据训练后再回填给同一训练集，否则会造成特征泄漏。
- 随机划分结果可能高估泛化能力，必须与化合物分组、物种分组划分一起报告。
- 土壤 `mg/kg` 主任务更贴近风险评估，但与摩尔尺度 QSAR 可比性不同，需要在论文中解释。
